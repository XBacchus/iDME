# iDME MiniApp 前端开发计划

> 设计规范请参考：[设计规范.md](./设计规范.md) | 设计模板预览：[设计模板.html](./设计模板.html)

## 一、项目概述

**技术栈**：Vue 3 + Vite + Axios
**开发目标**：完成工业 miniApp 四大核心功能模块的前端界面
**数据来源**：通过后端 API 调用 xDM-F 运行态接口

---

## 二、功能模块优先级（按评分权重）

### 🔴 优先级 1：物料管理（25分）
最复杂、分值最高，优先完成

### 🟠 优先级 2：设备管理（10分）
相对简单的 CRUD

### 🟡 优先级 3：工艺路线管理（10分）
需要关联工序，有一定复杂度

### 🟢 优先级 4：工序配置（5分）
最简单，仅展示固定数据

## 三、核心功能清单

### 模块 1：物料管理（25分）⭐ 重点

#### 1.1 物料列表页
**功能点**：
- 表格展示物料列表（编号、名称、规格型号、库存数量、供应商、分类、版本）
- 搜索功能：支持按物料编号/名称检索
- 分页功能
- 操作按钮：新增、编辑、删除、查看 BOM

**API 接口**：
```
GET /api/parts?search=xxx&page=1&size=10
```

#### 1.2 物料新增/编辑表单
**表单字段**：
- 物料编号（必填）
- 物料名称（必填）
- 规格型号
- 库存数量
- 供应商
- 分类（下拉选择或树形选择）
- 版本号

**API 接口**：
```
POST /api/parts
PUT /api/parts/{id}
```

#### 1.3 物料分类管理
**功能点**：
- 树形结构展示分类层级
- 支持新增/编辑/删除分类节点
- 物料创建时可选择分类

**API 接口**：
```
GET /api/parts/categories
POST /api/parts/categories
```

#### 1.4 版本管理
**功能点**：
- 查看物料历史版本列表
- 创建新版本（基于当前版本）
- 版本对比（可选加分项）

**API 接口**：
```
GET /api/parts/{id}/versions
POST /api/parts/{id}/versions
```

#### 1.5 BOM 管理（重点）
**功能点**：
- 树形结构展示 BOM 层级关系
- 添加子物料（选择物料 + 输入数量）
- 删除 BOM 关系
- 支持多层级展开/收起

**API 接口**：
```
GET /api/parts/{id}/bom
POST /api/parts/{id}/bom
DELETE /api/parts/{id}/bom/{childId}
```

### 模块 2：设备管理（10分）

#### 2.1 设备列表页
**功能点**：
- 表格展示设备列表（编码、名称、厂家、品牌、规格型号、位置）
- 操作按钮：新增、编辑、删除、查看详情

**API 接口**：
```
GET /api/equipments
```

#### 2.2 设备新增/编辑表单
**表单字段**：
- 设备编码（必填）
- 设备名称（必填）
- 生产厂家
- 品牌
- 规格型号
- 供应商
- 生产日期（日期选择器）
- 使用年限（数字输入）
- 折旧方式
- 位置

### 模块 3：工艺路线管理（10分）

#### 3.1 工艺路线列表页
**功能点**：
- 表格展示工艺路线（编号、名称、版本、所属产品）
- 操作按钮：新增、编辑、查看详情

**API 接口**：
```
GET /api/working-plans
```

#### 3.2 工艺路线详情页（重点）
**功能点**：
- 展示工艺基本信息（编号、名称、版本、所属产品、描述）
- 关联工序列表（显示顺序、工序名称）
- 支持拖拽排序工序顺序
- 添加/移除工序

**必须数据**：
- 工艺名称：`中心轮零件加工`
- 版本：`1.0`
- 关联 5 道工序：毛坯制造 → 粗加工 → 精加工 → 检测 → 入库

**API 接口**：
```
GET /api/working-plans/{id}
POST /api/working-plans/{id}/procedures
DELETE /api/working-plans/{id}/procedures/{procedureId}
```

#### 3.3 工艺路线新增/编辑表单
**表单字段**：
- 工艺编号（必填）
- 工艺名称（必填）
- 版本号（必填）
- 所属产品（必填）
- 工艺描述
- 操作人员
- 操作时间
- 设备使用情况

**API 接口**：
```
POST /api/working-plans
PUT /api/working-plans/{id}
```

---

### 模块 4：工序配置（5分）

#### 4.1 工序列表页
**功能点**：
- 表格展示固定 5 道工序
- 只读展示，不支持新增/删除
- 可编辑工序详细信息（生产步骤、操作人员等）

**固定数据**：
1. 毛坯制造
2. 粗加工
3. 精加工
4. 检测
5. 入库

**API 接口**：
```
GET /api/procedures
PUT /api/procedures/{id}
```

## 四、页面结构设计


- 企业级 UI 设计

```bash

```

### 方案 3：Naive UI
- 轻量级
- TypeScript 友好
- 现代化设计

```bash
npm install naive-ui
```

## 八、Mock 数据策略

### 8.1 开发阶段使用 Mock
在后端 API 未完成前，使用 Mock 数据：

```javascript
// src/api/mock.js
export const mockParts = [
  {
    id: 1,
    partCode: 'P001',
    partName: '中心轮',
    specModel: 'CL-100',
    stockQty: 50,
    supplier: '供应商A',
    categoryPath: '机械零件/轴类',
    versionNo: '1.0'
  }
]
```

### 8.2 切换真实 API
后端完成后，只需修改 `request.js` 的 `baseURL`：

```javascript
// 开发阶段
const baseURL = '/mock'

// 联调阶段
const baseURL = '/api'
```

## 十、注意事项

### 10.1 硬性要求
1. **工序名称必须完全一致**：毛坯制造、粗加工、精加工、检测、入库
2. **工艺路线必须包含**：名称"中心轮零件加工"、版本"1.0"
3. **所有数据通过后端 API**：不能直接调用 xDM-F API

### 10.2 加分项
1. 响应式设计（适配不同屏幕）
2. 操作确认提示（删除前二次确认）
3. 加载状态提示（Loading）
4. 数据导出功能（Excel）

### 10.3 避免踩坑
1. 不要过度设计，优先完成核心功能
2. 不要在 UI 美化上花费过多时间
3. 及时与后端对齐接口格式
4. 每完成一个模块就测试一次
