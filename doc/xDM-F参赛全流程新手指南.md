# xDM-F 参赛全流程新手指南（本科组）

## 1. 这份指南是给谁的

- 第一次参加工业 miniAPP 开发赛项
- 第一次使用华为云 iDME / xDM-F
- 需要一份“照着做就能完成”的操作手册

---

## 2. 最终你要交什么（先看结果）

按赛题要求，最终提交总文件夹（命名：`参赛编号-团队名-作品名`）至少包含：

1. 应用功能演示录屏  
2. 工程代码 zip  
3. xDM-F 模型导出 Excel + 模型图谱截图  
4. 工程中调用的 xDM-F API 列表

---

## 3. 总流程概览（先建立全局认知）

1. 账号与环境准备  
2. 进入 iDME，创建应用  
3. 在 xDM-F 创建 4 个核心模型  
4. 配置模型关系（工艺-工序、工序-设备、工序-物料、BOM）  
5. 录入赛题要求数据（5 道工序 + 1 条工艺路线）  
6. 导出模型与截图留证  
7. 本地部署运行态 SDK，后端统一调用 xDM-F API  
8. 开发前后端功能并录屏  
9. 打包并提交

---

## 4. 第 0 步：账号与环境准备

### 4.1 注册与登录

1. 注册华为云账号并登录控制台  
2. 打开 iDME 页面：  
   `https://console.huaweicloud.com/dme/?locale=zh-cn&region=cn-north-4#/dme/runenv`

### 4.2 如果出现“检测到您的设备不安全”

依次尝试：

1. 使用你本机手动打开的 Chrome（不要用自动化控制窗口）  
2. 切换网络（例如手机热点）  
3. 关闭代理/VPN  
4. 清理 `huaweicloud.com` 相关 Cookie 后重登  
5. 使用无痕窗口登录

### 4.3 本地环境安装（最低要求）

1. `JDK 17`（必选）  
2. `Maven`（建议）  
3. `Node.js`（前端用）  
4. 编辑器：IDEA / VSCode（任选）

---

## 5. 第 1 步：进入 iDME 并创建应用

1. 登录后进入 iDME 控制台  
2. 打开“设计服务 / 数据建模（xDM-F）”  
3. 点击“新建应用”（或进入已有应用）  
4. 记录以下信息（后续 License、SDK 需要）：  
   - 华为云账号 ID  
   - 应用 ID（applicationId）

---

## 6. 第 2 步：创建 4 个核心模型（重点）

> 建模原则：先创建模型，再加字段；每加完一批字段就保存一次。  
> 命名建议严格按赛题推荐，减少扣分风险。

### 6.1 模型一：设备 `Equipment`

创建模型：

- 中文名：设备  
- 英文名：`Equipment`

新增字段（建议）：

1. `equipmentCode` 设备编码（字符串 64，必填）  
2. `equipmentName` 设备名称（字符串 128，必填）  
3. `manufacturer` 生产厂家（字符串 128）  
4. `brand` 品牌（字符串 64）  
5. `specModel` 规格型号（字符串 128）  
6. `supplier` 供应商（字符串 128）  
7. `productionDate` 生产日期（日期）  
8. `serviceLifeYears` 使用年限（整数）  
9. `depreciationMethod` 折旧方式（字符串 64）  
10. `location` 位置（字符串 128）  
11. `technicalParams` 技术参数（长文本，扩展属性）  
12. `sparePartsInfo` 备品备件信息（长文本，扩展属性）

### 6.2 模型二：物料 `Part`

创建模型：

- 中文名：部件（或物料）  
- 英文名：`Part`

新增字段（建议）：

1. `partCode` 物料编号（字符串 64，必填）  
2. `partName` 物料名称（字符串 128，必填）  
3. `specModel` 规格型号（字符串 128）  
4. `stockQty` 库存数量（整数或小数）  
5. `supplier` 供应商（字符串 128）  
6. `categoryPath` 分类路径（字符串 256）  
7. `versionNo` 版本号（字符串 32）

### 6.3 模型三：工序 `WorkingProcedure`

创建模型：

- 中文名：工序  
- 英文名：`WorkingProcedure`

新增字段（建议）：

1. `procedureCode` 工序编号（字符串 64，必填）  
2. `procedureName` 工序名称（字符串 128，必填）  
3. `productionStep` 生产步骤（长文本）  
4. `operatorName` 操作人员（字符串 64）  
5. `startTime` 开始时间（日期时间）  
6. `endTime` 结束时间（日期时间）

### 6.4 模型四：工艺 `WorkingPlan`

创建模型：

- 中文名：工艺  
- 英文名：`WorkingPlan`

新增字段（建议）：

1. `planCode` 工艺编号（字符串 64，必填）  
2. `planName` 工艺名称（字符串 128，必填）  
3. `versionNo` 版本号（字符串 32，必填）  
4. `belongProduct` 所属产品（字符串 128，建议必填）  
5. `planDescription` 工艺描述（长文本）  
6. `operatorName` 操作人员（字符串 64）  
7. `operationTime` 操作时间（日期时间）  
8. `equipmentUsage` 设备使用情况（长文本）

### 6.5 本步完成判定

模型列表中能看到：

1. `Equipment`  
2. `Part`  
3. `WorkingProcedure`  
4. `WorkingPlan`

并且每个模型都已保存（若页面有“发布”按钮，再发布一次）。

---

## 7. 第 3 步：配置关系（重点）

在“关系管理 / 关联关系 / 参考对象”等入口创建以下关系：

1. `WorkingPlan -> WorkingProcedure`（工艺包含工序）  
2. `WorkingProcedure -> Equipment`（工序使用设备）  
3. `WorkingProcedure -> Part`（工序关联物料）  
4. `Part -> Part`（BOM 父子关系）

建议给关系加上业务字段（若页面支持）：

1. BOM 关系：`quantity`（数量）、`unit`（单位）  
2. 工艺-工序关系：`seqNo`（顺序号）

本步完成判定（新增）：

1. 关系实体列表可见 4 条核心关系  
2. 4 条关系状态均为“已发布”（不要停留在“正在工作”）

---

## 8. 第 4 步：录入赛题指定数据（拿分关键）

### 8.1 固定 5 道工序（必须完全一致）

1. 毛坯制造  
2. 粗加工  
3. 精加工  
4. 检测  
5. 入库

### 8.2 固定工艺路线（必须）

1. 工艺名称：`中心轮零件加工`  
2. 所属产品：按你的业务对象填写且保持前后一致（建议必填）  
3. 版本：`1.0`  
4. 工艺描述：建议明确写出流程背景与目标  
5. 正确关联上面 5 道工序并按顺序排列

说明：赛题里“工艺流程”描述为“毛坯制造→机械加工→轴类零件加工→尺寸检测→入库”，
在实现时可映射为固定工序“毛坯制造→粗加工→精加工→检测→入库”。

---

## 9. 第 5 步：导出与留证（提交必备）

1. 导出模型 Excel  
2. 截图模型图谱（至少包含 4 模型 + 关系）  
3. 导出或整理 API 调用清单（接口名、用途、请求方法、示例）

建议截图清单：

1. 模型列表页  
2. 关系图谱页  
3. 工序 5 条数据页  
4. 工艺路线详情页（显示版本 1.0 和工序关联）

---

## 10. 第 6 步：部署运行态 SDK 并联调

> 目标：让本地后端统一调用 xDM-F API。  
> 原则：业务层、展示层与数据层交互必须通过 xDM-F API。

1. 在 xDM-F 设计端下载运行态 SDK  
2. 按官方教程选择部署方式（Windows / Linux / 容器）  
3. 配置数据库与依赖（按官方文档）  
4. 启动运行态服务并做健康检查  
5. 后端先打通最小调用：新增 1 条设备、查询设备列表

---

## 11. 第 7 步：开发 miniApp 四大功能

1. 设备管理：增删改查 + 扩展属性  
2. 物料管理：增删改查 + 分类 + 版本 + 搜索 + BOM  
3. 工序管理：维护 5 道标准工序  
4. 工艺路线管理：中心轮零件加工 V1.0 + 工序顺序关联

实现建议：

1. 后端先封装统一 `xDM-F Client`，不要在业务代码里散落调用  
2. 前端优先做“可演示闭环”而非复杂美化  
3. 每完成一个模块立刻录一段短视频，防止最后来不及

## 15. 常见错误与避免方式

1. 错误：后端直接连自建数据库，不走 xDM-F  
   避免：所有业务读写统一走 xDM-F 运行态 API

2. 错误：工序名称写错或少一条  
   避免：直接复制标准名称，录入后逐字核对

3. 错误：只建模型，不建关系  
   避免：把“关系配置”单独当成一个里程碑

4. 错误：最后一天才录屏和导出材料  
   避免：每完成一个模块就留证
