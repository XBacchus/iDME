### 2.3 模型关键字段怎么检查（你现在最需要看这个）

按下面路径逐个核验：

1. 进入 `xDM-F 设计态 -> 数据模型`
2. 点击模型（`Equipment/Part/WorkingProcedure/WorkingPlan`）
3. 打开“字段/属性”列表，核对：`字段中文含义`、`字段类型`、`是否可录入`
4. 点击“发布状态”确认是“已发布”

字段判定口径（按赛题评分最小要求）：

#### A. `Equipment`（设备）
- [ ] 已有字段覆盖：设备编码、设备名称、生产厂家、品牌、规格型号、供应商、生产日期、使用年限、折旧方式、位置
- [ ] 建议类型合理：编码/名称等为文本，生产日期为日期，使用年限为数值
- [ ] 扩展信息可承载：技术参数、备品备件（文本或长文本均可）

#### B. `Part`（物料）
- [ ] 已有字段覆盖：物料编号、物料名称、规格型号、库存数量、供应商
- [ ] 建议类型合理：库存数量为数值，其余多为文本
- [ ] 后续可支撑分类与版本（字段或关联方式都可）

#### C. `WorkingProcedure`（工序）
- [ ] 已有字段覆盖：工序编号、工序名称、生产步骤、操作人员、开始时间、结束时间
- [ ] 建议类型合理：开始/结束时间为日期时间

#### D. `WorkingPlan`（工艺）
- [ ] 已有字段覆盖：工艺编号、工艺名称、操作人员、操作时间、设备使用情况
- [ ] 为功能评分建议补齐：所属产品、版本、工艺描述（用于工艺路线10分项）

判定说明：

- [ ] 字段英文名不强制完全一致，但中文业务含义必须能一一对应赛题要求
- [ ] 英文模型名强烈建议与赛题推荐一致：`Equipment/Part/WorkingProcedure/WorkingPlan`

### 2.4 关系实体怎么检查（避免“发布了但方向错了”）

按下面 6 步验收，每一步都要过：

1. **定义检查**  
   在“关系实体/关联关系”列表确认存在 4 条关系：  
   `WorkingPlan -> WorkingProcedure`、`WorkingProcedure -> Equipment`、`WorkingProcedure -> Part`、`Part -> Part`

2. **方向检查（最容易错）**  
   打开每条关系详情，确认“源模型 -> 目标模型”方向与上面一致。  
   例如：工序用设备必须是 `WorkingProcedure -> Equipment`，不是反向。

3. **发布状态检查**  
   4 条关系状态均为“已发布”；只要有“正在工作/未发布”，按未完成处理。

4. **实例可用性检查（关键）**  
   不只看定义，必须新增一条关系数据验证：  
   - 给某条 `WorkingPlan` 关联 5 道 `WorkingProcedure`  
   - 给某条 `WorkingProcedure` 关联设备和物料  
   - 给两条 `Part` 建立一条 BOM 父子关系

5. **查询回读检查**  
   从“工艺详情页”能回读到完整工序链；  
   从“工序详情页”能回读到设备和物料；  
   从“物料/BOM页面”能看到父子关系。

6. **图谱与导出检查**  
   在模型图谱中看到连线；  
   在模型导出 Excel 中看到模型与关系定义（作为提交证据）。

关系错误判定（任一命中即不通过）：

- [ ] 关系存在但方向错误
- [ ] 关系已发布但无法创建关系数据
- [ ] 工艺路线未关联全 5 道工序
- [ ] BOM 只能建定义，不能实际挂接 Part- Part

- [ ] `Equipment` 为实体模型（不是抽象）
- [ ] `Part` 为实体模型（不是抽象）

- [ ] 赛题要求要新增、修改、查询真实业务数据，抽象模型通常不能直接承载业务实例
