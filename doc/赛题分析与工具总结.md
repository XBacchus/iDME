# 工业miniAPP赛题分析与工具总结（本科组）

## 1. 这道赛题到底要你做什么
一句话：
基于华为 iDME 的 xDM-F 做完整的数据建模，并开发一个工业 miniApp，把“设备、物料(BOM)、工序、工艺路线”这套制造管理流程跑通，且前后端与数据层交互必须走 xDM-F API。

## 2. 你必须完成的功能（按任务拆解）
### 2.1 设备管理
- 实现设备基础信息的新增、修改、删除、查询（CRUD）。
- 基础字段至少包含：设备编码、设备名称、生产厂家、品牌、规格型号、供应商、生产日期、使用年限、折旧方式、位置。
- 支持扩展属性：技术参数信息、备品备件信息等。

### 2.2 物料 Part + BOM 管理
- 维护 Part 属性（如物料编号、名称、规格型号、库存数量、供应商等）。
- 支持物料分类管理（层级分类）。
- 支持版本管理。
- 支持按编码/名称检索。
- 支持构建 BOM（Part 之间组成关系）。

### 2.3 配置工序（固定5道）
按题目要求配置以下工序：
1. 毛坯制造
2. 粗加工
3. 精加工
4. 检测
5. 入库

### 2.4 配置工艺路线
- 新增一条工艺路线：`中心轮零件加工`。
- 版本：`1.0`。
- 工艺流程围绕：毛坯制造 -> 机械加工 -> 轴类零件加工 -> 尺寸检测 -> 入库。
- 需要正确关联工序。

### 2.5 工业数据建模（xDM-F 中完成）
至少建好以下模型并可用：
- 设备模型：`Equipment`
- 物料模型：`Part`
- 工序模型：`WorkingProcedure`
- 工艺模型：`WorkingPlan`

还要建好关键关系：
- 工艺 <-> 工序
- 工序 <-> 设备
- 工序 <-> 物料

并可按需补充其他支撑 miniApp 的模型。

## 3. 硬性约束（不满足会直接影响得分）
- 业务层、展示层、数据层交互必须通过 xDM-F API；绕过 xDM-F 计 0 分（对应项）。
- 数据建模在 xDM-F 公有云完成；建模后下载运行态 SDK，本地部署后由业务层调用本地运行态 API。
- 必选环境：`xDM-F`、`JDK17`。
- 本科组还强调：需求优化、多场景适配、性能验证。
- 说明文件还要求作品需基于国产大模型（二次开发/集成也可）或使用了 AI 技术，并提供说明材料。

## 4. 你最终要交什么
提交总文件夹命名：`参赛编号-团队名-作品名`。

至少包含：
1. 应用功能演示录屏
2. 工程代码 zip
3. iDME xDM-F 模型导出 Excel + 模型图谱截图
4. 工程中调用的 iDME xDM-F API 列表

## 5. 评分怎么拿（100分拆解）
| 评分项 | 分值 | 关键点 |
|---|---:|---|
| 设备管理 | 10 | 字段正确 + 扩展属性支持 |
| 物料管理 | 25 | 新增正确 + 版本管理 + 分类管理 + 搜索 + BOM |
| 配置工序 | 5 | 5道工序名称与数量正确 |
| 配置工艺路线 | 10 | 工艺基础信息正确 + 工序关联正确 |
| 工业数据建模 | 30 | 模型创建正确 + 关系创建正确 |
| 架构先进性 | 10 | 可配置、可扩展（主观） |
| 用户体验 | 10 | 界面、响应、操作（主观） |
| 扣分项 | - | 违纪等 |

优先级建议（冲分顺序）：
1. 先拿客观高分项：工业数据建模(30) + 物料管理(25)
2. 再做设备管理(10) + 工艺路线(10) + 工序(5)
3. 最后打磨架构先进性(10)和用户体验(10)

## 6. 这题会用到哪些工具（汇总）
### 6.1 必选/核心
- 华为云 `iDME / xDM-F`
- `xDM-F 运行态 SDK`（本地部署后给后端调用）
- `JDK 17`

### 6.2 官方给出的开发栈（可选）
- 前端：Node.js、VSCode、HTML5/CSS/JS/Vue
- 后端：Maven、IntelliJ IDEA、Spring Boot 3.0

### 6.3 附件中的工具与资源
- 软件清单：工业数字模型驱动引擎 iDME（华为云控制台）
- 视频教程：
  - Linux 部署运行 SDK
  - Windows 部署运行 SDK
  - 容器部署运行 SDK
  - Java 工程集成运行 SDK
- SDK 参考文档：iDME SDK 参考（含 Open API、运行 SDK、流程 SDK、常见问题）

### 6.4 SDK 部署常见依赖（按文档提炼）
- 部署模式：Linux / Windows / 容器 / Java 工程集成
- 数据库（必需其一）：MySQL 5.x/8.x、PostgreSQL 13.x/14.x、GaussDB（对应版本）
- Redis：5.x/6.x（常用必需）
- MongoDB：可选（不配会影响部分功能，如系统版本能力）
- 配置中心可选：Nacos 或 ServiceComb
- License：需要按应用ID和机器指纹申请（Windows 用 MAC，Linux 用 UUID）

## 7. 建议你按这个节奏做
1. 在 xDM-F 先把 4 个核心模型和关系建好并发布。
2. 下载运行态 SDK，本地先跑通健康检查和 API 调用。
3. 后端先做设备/物料/工序/工艺路线 API 封装（统一走 xDM-F）。
4. 前端先做最小可用页面：设备管理、物料管理、工序配置、工艺路线配置。
5. 补齐物料分类、版本、BOM、搜索这些 25 分大项细节。
6. 导出模型 Excel + 截图，整理 API 清单。
7. 录制演示视频，整理代码包与最终提交目录。
