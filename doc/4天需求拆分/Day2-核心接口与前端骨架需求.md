# Day2 需求文档：核心接口与前端骨架联通

## 1. 当日目标

完成“可联调最小闭环”：

1. 四大模块后端 CRUD 基本可用。
2. 前端页面完成列表 + 新增 + 编辑 + 删除的主流程。
3. 工艺路线与工序关联能力可演示。
4. API 文档同步第一版。

## 2. 并行任务包

### 任务包 A（后端设备与工序）

范围：

1. 设备管理 API：增删改查。
2. 工序管理 API：增删改查（包含固定 5 道工序初始化能力或校验）。

输出：

1. 接口实现与异常处理。
2. 接口请求示例与返回示例。

验收标准：

1. Postman/Apifox 可调通。
2. 错误输入能返回明确错误信息。

### 任务包 B（后端物料/BOM 与工艺）

范围：

1. 物料 API：增删改查、按编码/名称查询。
2. BOM API：`Part -> Part` 关系维护。
3. 工艺路线 API：创建“中心轮零件加工 V1.0”并关联工序。

输出：

1. 业务接口实现。
2. 关系写入与读取能力。

验收标准：

1. 能读取完整工艺-工序链路。
2. BOM 关系可创建可查询。

### 任务包 C（前端页面主流程）

范围：

1. 设备、物料、工序、工艺路线四页面主流程。
2. 表单校验与基础反馈。

输出：

1. 页面可操作闭环。
2. 与后端接口联通。

验收标准：

1. 每个模块至少完成 1 次新增和 1 次查询演示。
2. 页面有空态和失败提示。

### 任务包 D（API 文档与联调）

范围：

1. 更新 `API_DOCS`（需求与实现闭环）。
2. 记录联调问题单并回归关闭。

输出：

1. API 文档第一版。
2. 联调问题记录。

验收标准：

1. 接口文档可覆盖已实现接口。
2. 文档与真实返回一致。

## 3. 串行依赖

1. 任务包 A/B 接口稳定后，任务包 C 再进行批量联调。
2. 文档任务包 D 需在 A/B 接口冻结后收口。

## 4. 当日产物清单

1. 四模块 API 清单 v1。
2. 前端联调演示视频（短片段即可）。
3. Day2 问题清单与剩余风险。
