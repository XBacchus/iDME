# Day3 需求文档：功能完善与全链路联调

## 1. 当日目标

完成“评分核心项”：

1. 物料分类、版本、搜索、BOM 深化。
2. 工艺路线顺序编排与回读稳定。
3. 前后端全链路回归。
4. 验收清单覆盖率达到可提交水平。

## 2. 并行任务包

### 任务包 A（后端增强）

范围：

1. 物料分类与版本管理接口补齐。
2. 搜索条件组合查询优化。
3. 工艺路线与工序顺序字段（如 `seqNo`）完善。

输出：

1. 增强接口与查询性能优化点。

验收标准：

1. 物料 25 分项关键点全部可演示。
2. 工艺顺序读取与保存一致。

### 任务包 B（前端增强）

范围：

1. 物料页支持分类筛选、版本展示、搜索入口。
2. 工艺路线页支持顺序展示与编辑。
3. 页面交互可用性优化（提示文案、禁用态、异常态）。

输出：

1. 增强交互页面。

验收标准：

1. 操作路径清晰，错误可恢复。
2. 关键页面可稳定演示。

### 任务包 C（测试验收）

范围：

1. 按验收清单执行全量检查。
2. 对照 `miniapp/backend/tests/TEST_ACCEPTANCE_CHECKLIST.md` 补齐状态标记。

输出：

1. 覆盖状态标记（✅/⚠️/❌）。
2. 缺陷清单和优先级。

验收标准：

1. 核心场景至少完成一轮“三段式”验证：设计输入 -> 设计预期 -> 输出对比。

### 任务包 D（文档与证据）

范围：

1. 整理模型导出、图谱截图、API 列表草稿。
2. 同步 `_verify/api-docs.json` 快照。

输出：

1. 提交材料草稿版。

验收标准：

1. 材料真实、可追溯、可复验。

## 3. 串行依赖

2. C 通过后，D 再做材料冻结。

## 4. 当日产物清单

1. 功能完成度报告（按评分项）。
2. 缺陷关闭率报告。
3. 提交材料草稿包。
