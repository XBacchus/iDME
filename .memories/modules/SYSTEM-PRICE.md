# SYSTEM-PRICE（系统口径基线，历史命名保留）

## 目的
统一 iDME 关键口径，避免前后端、网关代理、流程规则出现多套定义。

## 原则
1. 单一主数据源：同一业务指标只允许一个主数据源。
2. 字段与规则变更必须可追溯（来源、时间、版本）。
3. API 输入/输出口径必须可对账、可回放。
4. 若出现特例口径，必须在 API 文档和测试清单中显式声明。

## 变更要求
- 涉及核心口径、规则计算或接口契约时，必须同步更新：
  - `API_DOCS/*`
  - `_verify/api-docs.json`
  - `miniapp/backend/tests/TEST_ACCEPTANCE_CHECKLIST.md`
  - 本文档（如口径有变化）
